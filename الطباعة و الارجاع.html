الطباعة و الارجاع

حتى الان الالة الحاسبة التي بنيناها تعمل وفي درس اليوم سوف نعرف الفرق بين طباعة شيء ما في ال console وارجاع قيمة من دالة كمخرج

مثلا في دالة الجمع لماذا علي ان ارجع قيمة الجمع لما لا اقوم بطباعتها وفقط؟ وهذا ما سوف نجيب عليه في درس اليوم

هذا هو الكود الذي كتبناه في الدرس السابق: 


def add(n1, n2):
    return n1 + n2


def subtract(n1, n2):
    return n1 - n2


def multiply(n1, n2):
    return n1 * n2


def divide(n1, n2):
    return n1 / n2


operations = {
    "+": add,
    "-": subtract,
    "*": multiply,
    "/": divide,
}

num1 = int(input("What's the first number?: "))

for symbol in operations:
    print(symbol)
operation_symbol = input("Pick an operation from the line above: ")

num2 = int(input("What's the second number?: "))

calculation_function = operations[operation_symbol]
answer = calculation_function(num1, num2)
print(f"{num1} {operation_symbol} {num2} = {answer}")

في حالته الان لا يمكنك ملاحظة الفرق بين استعمال print او return

اليك السيناريو التالي: ماذا لو سألت المستخدم ان يختار عملية اخرى و رقم ثالث اخر لكي يكمل العملية من النتيجة السابقة ثم تمرر العدد الثالث مع القيمة السابقة للدالة calculation_function والتي هي first_answer والعدد الثالث وهو num3 هكذا: 


operation_symbol = input("Pick another operation: ")
num3 = int(input("What is the next number?: "))
calculation_function = operations[operation_symbol]
second_answer = calculation_function(first_answer, num3)

لاحظ هنا انني سوف اغير اسم المتغير answer في المرة الاولى اسميه first_answer وفي المرة الثانية second_answer حتى لا يختلط الاثنين ثم اقوم بطباعة النتيجة 


print(f"{first_answer} {operation_symbol} {num3} = {second_answer}")

الطباعة هي مجرد رسالة تراها في الشاشة console ولا يمكنك استخدامها داخل البرنامج الخاص بك بينما عندما ترجع القيمة ك مخرج للدالة يمكنك تخزينها في متغير وتمريرها للدوال وفعل مختلف الاشياء مع تلك القيمة.
